 Besides Unicode text, this project also contains templates. A template is a function, often with one or more parameters, that represents visible features in the masoretic text or concepts related to its presentation. These features or concepts require documentation and/or formatting beyond the simple Unicode characters that are used for letters, niqqud and cantillation marks.
Templates are always contained with double curly brackets like this: {{template}}. The first component in a template always its name, which defines its function. Following the name may be one or more required parameters, optional parameters, and/or documentation notes.
Each distinct component within a template (name, parameters, notes) is separated within the template by a horizontal line ("|") like this:
{{name}}
or
{{name|parameter}}
or
{{name|parameter|note(s)}}
or
{{name|parameter1|parameter2|note(s)}}.
But in Hebrew these components are of course typed from right to left like this:
{{שם|משתנה1|משתנה2|הערות}}.
To see exactly how a template functions in Wikisource, click on its link in this list and then go to its edit page to examine the code.
Any implementation of this project, whether at Wikisource or elsewhere, must decide how to best interpret and apply the templates found within the text (and listed here) according to its particular needs and goals. It is *not* recommended to simply copy the text as it is.
Note that the names of many of these templates begin with the prefix "מ:" which is short for "מקרא". This prefix indicates that the template is specifically designed for use within the Miqra_al_pi_ha-Mesorah project, and is not meant for general use throughout Wikisource. Some of the other templates will also be changed to names with this prefix in the future.		Original list - רשימה מקורית מלבד טקסט מוקלד ביוניקוד, יש בפרויקט הזה גם תבניות. תבנית היא פונקציה, לעתים קרובות עם משתנה אחד או יותר, המייצגת מאפיינים הנראים לעין בתוך נוסח המסורה או מושגים הקשורים לנוסח המסורה. המאפיינים והמושגים הללו חמייבים תיעוד ו/או עיצוב מעבר לתווים הפשוטים ביוניקוד שמשתמשים בהם בשביל האותיות והניקוד והטעמים.
תבניות באות תמיד בתוך סוגריים מסולסלים כך: {{תבנית}}. המרכיב הראשון בכל תבנית הוא שמה, המגדיר את הפונקציה שלה. לאחר השם יכולים לבוא משתנה חובה אחד או יותר, משתנים אופציונליים, ו/או הערות תיעוד.
כל מרכיב ייחודי בתוך תבנית (שם, משתנים, הערות) מובדל על ידי קו אנכי ("|") כך:
{{שם}}
או
{{שם|משתנה}}
או
{{שם|משתנה|הערה או הערות}}
או
{{שם|משתנה1|משתנה2|הערה או הערות}}.
כדי לראות איך בדיוק התבנית פועלת בוויקיטקסט, יש ללחוץ על הקישור שלה ברשימה זו ולפתוח את דף העריכה שלה בוויקיטקסט כדי לראות את הקוד שלה.
כל יישום של הפרויקט הזה, בוויקיטקסט או במקום אחר, חייב אם כן להחליט על הדרך הראויה ביותר לפרש וליישם את התבניות הנמצאות בתוך הטקסט (והמופיעות ברשימה זו), לפי הצרכים והמטרות המיוחדים לאותו יישום. *לא* מומלץ להעתיק את הטקסט כמו שהוא.
כדאי לשים לב שהשמות ברבות מהתבניות הללו מתחחילים ב"מ:" שהוא קיצור ל"מקרא". תוספת זו בתחילת השם מציינת שהבתנית עוצבה במיוחד עבור שימוש בפרויקט "מקרא על פי המסורה", ואינה מיועדת לשימוש כללי בוויקיטקסט. גם השמות בחלק משאר התבניות ישתנו בעתיד בתוספת "מ:" בתחילתם.

The documentation template:תבנית התיעוד:
4
This is the documentation template (נוסח), which is used extensively. It has a single parameter (משתנה), namely the exact text that is to be shown. Following the text itself are notes (הערות) that document and discuss anomolous or surprising forms, variant readings, uncertain readings, and other data relevant to the particular text at hand. The notes do not have any effect on how the template functions. The purpose of this template is to mark off notable or unusual or questionable elements within the text, and its notes serve to document any local data relevant to them. But the template itself is designed not to interfere in any way with how the text appears to the reader.{{נוסח|משתנה|הערות}}זוהי התבנית לתיעוד הנוסח, שהשימוש בה נפוץ ביותר. יש בה משתנה אחת, שהיא הטקסט המדוייק המופיע בפועל בתוך המהדורה. לאחר הטקסט עצמו יש בתוך התבנית הערה אחת או כמה הערות המתעדות שינויי גירסה מובהקים, שינויי גירסה מסופקים, גרסאות מתהימות, וכל מידע נוסף שהוא רלוונטי לקביעת הנוסח במקום הנתון. בסך הכל יש בתבניות הנוסח ציוּן ותיעוד מלא לכל פרט, ולו הקטן ביותר, שיכול להשפיע על קביעת הנוסח במהדורתנו (באותיות ובניקוד ובטעמים ובחלוקת הפרשות ובתחומים נוספים). אין להערות הללו שום השפעה על תפקודה של התבנית בפועל. מטרתה של התבנית היא אך ורק לתייג את המרכיבים בטקסט הראויים לציון, או שהם חריגים או מסופקים, וההערות במקומות הללו באות רק כדי לספק מידע לגביהם. לכן התבנית "נוסח" בנויה כדי שלא להפריע כלל, בכל דרך שהיא, על הצגת הטקסט בפני הקורא.

Navigation templates (mostly in Column D):תבניות ניווט (בעמודה D בעיקר):
6
This is the main navigation template, which appears immediately before every single verse in Column D. It has three required parameters that must be used for every verse: The first (משתנה1) is book name, the second (משתנה2) is chapter number, and the third (משתנה3) is verse number. It also has optional parameters for seder number (סדר=) and aliyah number in the Torah (עלייה=). Further optional parameters can easily be added in the future. The template creates visible numbering and labeling in the right margin (for chapter and verse numbers) and in the left margin (for seder and aliyah numbers). Note that all chapter and verse numbering in this edition follows Koren (the only exception to this rule being for the initial verses of the Ten Commandments), which is a good representative of the typical numbering in Jewish editions. Variations in verse numbering are sometimes noted by using the documentation template (in such cases this template itself becomes the variable within the documentation template in Column D).{{מ:פסוק|משתנה1|משתנה2|משתנה3|סדר=|עלייה=}}זוהי תבנית הניווט העיקרית, המופיעה לפני כל פסוק בטור D. יש בו שלושה משתנים שהם חובה בכל פסוק: השם של הספר, מספר הפרק, ומספר הפסוק (כולם באותיות עבריות). יש בה גם משתנים אופציונליים למספר הסדר ("סדר=") ומספר העלייה בתורה ("עלייה="). ניתן בקלות להוסיף בה משתנים אופציונליים נוספים לפי הצורך בעתיד. התבנית יוצרת בפועל סימני ניווט בשוליים בצד ימין (למספרי פרקים ופסוקים) ובצד שמאל (למספרי סדרים ועליות בתורה). יש לציין שכל מספרי הפרקים והפסוקים במהדורה זו הם תמיד לפי דפוס קורן (היוצא מן הכלל היחיד הוא בפסוקים הראשונים שבעשרת הדברות בלבד); דפוס קורן הוא נציג נאמן למספרים ברוב רובם של הדפוסים היהודיים, ובמיוחד בדורות האחרונים. אבל לפעמים גם ציינו הבדלים בחלוקת הפרקים (ובעקבותיהם גם במספור הפסוקים) ע"י השימוש בתבנית לתיעוד הנוסח (כאשר תבנית הניווט עצמה הופכת להיות המשתנה בתוך תבנית "נוסח" בטור D).

General formatting:עיצוב כללי:
11
This template provides a "title" at the top of every page in this edition with convenient links to tables of content, introductions, and practical information about fonts for viewing texts with cantillation.{{תבנית:טעמי המקרא באינטרנט}}תבנית זו יוצרת "כותרת" בראש כל דף במהדורה, ובכותרת זו יש קישורים נוחים לדפים שימושיים, כגון: תוכן העניינים, פרקי מבוא, ומידע על גופנים לטעמי המקרא.
12
This template sets the fonts and style for the texts in this edition which contain niqqud and cantillation. It has an optional parameter which is a number indicating the size of the font. The default size in Wikisource (if there is no number) is 23pt and it almost always appears this way. But there is an exception: For "song" format within the 21 books (which is formatted within tables), the size has been set to 18 in Wikisource by adding "18" as an optional parameter. However, normally there is no parameter.{{תבנית:משתמש:Dovi/טעמי המקרא}}תבנית זו מגדירה את הגופנים והעיצוב עבור הטקסטים במהדורה זו שיש בהם ניקוד וטעמים. יש בה משתנה אופציונלי אחד שהוא מספר המגדיר את גודל הגופן. ברירת המחדל בוויקיטקסט (אם אין מספר למשתנה זה) היא 23pt, וכך הוא ברוב המכריע של המקרא בווויקיטקסט. אבל יש לציין יוצא מן הכלל אחד: בצורת השיר בכ"א הספרים (המעוצבת בתוך טבלאות), גודל הגופן הוגדר ל-18pt בוויקיטקסט על ידי הוספת המספר "18" למשתנה האופציונלי. אמנם בדרך כלל אין משתנה.

Templates for the beginning of books:תבניות לראשי ספרים:
17
This template marks the precise beginning of the 24 books in the masoretic corpus, each of which is preceeded by defined spacing according to the masoretic tradition. It has a single parameter for the name of the book (e.g. שמואל). It is not used for 2 Samuel, 2 Kings, Nehemiah, 2 Chronicles, nor within the 12 minor prophets.{{מ:ספר חדש|משתנה}}תבנית זו מציינת את מקום ההתחלה המדוייק של הטקסט בכל אחד מכ"ד הספרים במקרא לפי המסורה, אשר לפני כל אחד מהם יש רווח מסויים במסורה. יש בו משתנה אחד לשמו של הספר (לדוגמה: "שמואל"). אין שימוש בתבנית זו בתחילת שמ"ב, מל"ב, דה"ב, נחמיה, וגם לא בתוך שנים עשר הנביאים "הקטנים" (כלומר בנביאי "תרי עשר").
18
This template marks the precise beginning of each of the 12 minor prophets; each of them is preceeded by defined spacing in the masoretic tradition. It has a single parameter for the name of the prophet (e.g. חבקוק).{{מ:רווח בתרי עשר|משתנה}}תבנית זו מציינת את התחלתו המדוייקת של כל אחד מנביאי תרי עשר, שיש ביניהם עיצוב של רווח מסויים לפי שיטת המסורה. בתבנית יש משתנה אחר עבור שם הנביא, לדוגמה: {{רווח בתרי עשר|חבקוק}}.

Templates for ketiv and qeri:תבניות לכתיב וקרי:
21
This is the main template for "ketiv and qeri". It has two required parameters, namely the ketiv (first) and the qeri (second). In Wikisource it is set to show the ketiv in gray first, followed by the ketiv (after a space in regular color). If there are textual anomolies associated with the ketiv/qeri pair at hand, then this entire template will be contained inside the documentation template as its parameter, followed by a note explaining the issue.{{כו"ק|משתנה1|משתנה2}}זאת התבנית הרגילה ל"כתיב וקרי". יש בה שני פרמטרים שהם חובה, דהיינו הכתיב (משתנה1) והקרי (משתנה2). תבנית זו מעוצבת בוויקיטקסט כדי להראות קודם כל את הכתיב באות אפורה, ואחר כך את הכתיב (באות רגילה לאחר רווח פשוט). אם יש תופעות חריגות לגבי צמד הכתיב/קרי במקום מסוים, אז תופיע התבנית הזאת במלואה בתוך תבנית התיעוד (ראו למעלה "תבנית:נוסח") בתור המשתנה היחיד, ולאחריה תבוא הערת נוסח שמסבירה את החריגה.
22
This template works exactly like the previous template, except for one thing: It shows the qeri (second parameter) *before* the ketiv (first parameter). This is necessary for purposes of formatting when the ketiv/qeri pair occurs immediately following maqaph.{{קו"כ|משתנה1|משתנה2}}תבנית זו עובדת בדיוק כמו התבנית הקודמת, חוץ מדבר אחד: היא מעוצבת כד להראות את קרי (משתנה2) *לפני* הכתיב (משתנה1). זה נחוץ בשביל עיצוב מתאים לטקסט כאשר צמד ההכתיב/קרי בא מיד לאחר מקף.
23
This template is used to mark ketiv/qeri pairs where the qeri is nothing more than a matter of matres lectionis (אֵם קריאה). In such cases this edition (like other editions such as Koren, Breuer) does not display a ketiv/qeri pair. It is nevertheless tagged by this template, which following its single parameter (containing the vocalized ketiv text to be shown in regular color) contains a note documenting the qeri (or sometimes yetir or haser) that appears in the manuscripts.{{קו"כ-אם|משתנה|הערה}}תבנית זו מתייגת צמדי כתיב/קרי במקומות שבהם הקרי הוא רק עניין של "אֵם קריאה" בלבד. במקומות כאלה לא מופיע במהדורתנו צמד של כתיב/קרי (בדומה למהדורות אחרות כגון קורן וברויאר). בכל זאת יש תיוג במקומות האלה על ידי התבנית הזאת, כי לאחר המשתנה היחיד בה (דהיינו אותיות הכתיב בניקוד ובטעמים שמופיע בטקסט באות רגילה), בא הערת נוסח המתעדת את הקרי (או לעתים את הערת ה"יתיר" או ה"חסר") שמופיע בכתבי היד.
24
This template tags and formats the special phenomenon of "ketiv vo-lo qeri" (a word written but not read). Its single parameter is the ketiv. In Wikisource the "ketiv" parameter is set so that it appears in gray, plus it is entered within parenthesis. For example: a parameter of "(אם)" in Ruth 3:12.{{כתיב ולא קרי|משתנה}}תבנית זו מתייגת ומעצבת את התופעה המיוחדת של "כתיב ולא קרי". המשתנה היחיד שבה הוא הכתיב. התבנית מעוצבת בוויקיטקסט כדי שהכתיב שאינו נקרא יופיע באות אפורה, ובנוסף לכך מכניסים אותו בתור משתנה בתוך סוגריים עגולים. לדוגמה: "(אם)" בתור משתנה ברות ג,יב.
25
This template tags and formats the special phenomenon of "qeri ve-lo ketiv" (a word read but not written). Its single parameter is the qeri. In Wikisource the "keri" parameter is set to appear like the rest of the biblical text in regular color, but it is entered between square brackets. For example: a parameter of "[אֵלַ֔י]" in Ruth 3:17.{{קרי ולא כתיב|משתנה}}תבנית זו מתייגת ומעצבת את התופעה המיוחדת של "קרי ולא כתיב". המשתנה היחיד שבה הוא הקרי. התבנית מעוצבת בוויקיטקסט כדי שהקרי יופיע כמו שאר נוסח המקרא באות רגילה, אבל מכניסים אותו בתור משתנה בתוך סוגריים מרובעים. לדוגמה: "[אֵלַ֔י]" בתור משתנה ברות ג,יז.

29
This is the common template representing an "open" parashah division (parashah petuhah). In Wikisource it is set to leave a blank line between the end of the previous parashah and the beginning of the next one (this format is a normal halakhic one according to Maimonides, based on the Tiberian masoretic manuscripts, and is still found in some Torah scrolls). This template and all the rest of the "parashah" templates have no parameters.{{פפ}}זוהי התבנית הרגילה עבור "פרשה פתוחה".
30
This is a variation on the previous template. At times it is appropriate to document an "open" parashah (petuhah) but for it to appear in a slightly different format, namely for the parashah to start at the beginning of the very next line (with no blank line in between).{{פפפ}}
31
This template supports the previous one by attaching some large blank spaces to the very end of the previous parashah, so that the text cannot continue until the end of the line (which halakhically allows the next parashah to begin on the very next line with no blank line in between). In Wikisource this template is set to show four sequential large blank spaces (" "), which are placed at the end of the line.{{רווח בסוף שורה}}
32
This is the common template representing a "closed" parashah division (parashah setumah). In Wikisource it is set to have the parashah begin on the very next line following an indentation (this format is a common halakhic one according to Maimonides, based on the Tiberian masoretic manuscripts, and is still found in some Torah scrolls). The indentation is formed through the use of three large blank spaces of this type: "　".{{סס}}
33
This is a variation on the previous template. At times it is appropriate to document a "closed" parashah (setumah), but for it to appear in a different format, namely for the parashah to start following a blank space in the middle of the line (with text before and after it). In Wikisource this template is set to show two consecutive large blank spaces ("　") that are "glued" to each other and to the text before and after them.{{ססס}}
34
This template is a variation on {{סס}}. It to shows a "closed" parashah (setumah) that begins on the next line following an indentation. But the indentation in this template is narrower, and in Wikisource it is set to show only two large blank spaces (" "). It is meant for use within text presented in relatively narrow columns (such as in charts that show the Ten Commandments in parallel columns).{{סס2}}

Sometimes the spacing for parashah divisions occurs within a verse (rather than before one), a phenomenon known as "פסקא באמצע פסוק". This template tags such occurences. Its first and most important parameter contains the parashah template itself, which indicates how the parashah spacing will appear within the text. A second parameter designates the verse (book, chapter and verse number). At times this is followed by a note about how the parashah division within the verse appears visually within one or more manuscripts.{{פסקא באמצע פסוק|משתנה|הערות}}לפעמים יש רווח של פרשה באמצע פסוק (במקום לפני פסוק), תופעה שנקרא "פסקא באמצע פסוק". תבנית זו מתייגת את התופעות הללו. המשתנה הראשון והחשוב בו מכיל את התבנית לצורת הפרשה, שקובעת כיצד תראה הרווח של הפרשה בטקסט (באמצע הפסוק). המשתנה השני מציין את הפסוק (ספר, פרק ומספר הפסוק). לפעמים יש אחר כך הערה על צורת הפרשה באמצע הפסוק בכתב יד אחד או יותר.

The chapter divisions are largely designed to mark where new sections begin. Therefore, the majority of chapters begin at points where visible parashot also begin. However, it is not infrequent for a chapter to begin where there is no new parashah. When the biblical text is presented sequentially, a space must be added before the first verse in the chapter since there is no parashah (but when the chapter is presented in isolation no such space is required). This template tags such chapters so that they can be properly formatted in the 21 books (see the next template for the three poetic books). It is placed in Column C of the first verse in the chapter.{{תבנית:מ:אין פרשה בתחילת פרק}}

45
In the masoretic tradition, the word "Yerushalayim" is almost always written without the letter yod (ירושלם). In practical terms this means that three diacritical marks need to be entered after the letter lamed: Its own vowel (patah or qamaz) and cantillation note, plus the vowel of the missing yod (namely hiriq). This combination does not always format and display well. To solve the display problem, this template records the vowel and cantillation of the lamed as its two parameters, plus it automatically provides the hiriq of the missing yod while adding the Unicode character ZWJ (zero width joiner) between the diacriticals so that they display properly. For rare cases where the form of the word is different, and this automatic template cannot be used, the ZWJ has been added to the text manually and this has been noted in a documentation template; see 2 Kings 9:28, Isaiah 36:2, and Ezekiel 8:3. (In 2 Chronicles 32:9 the word is written fully with the letter yod and there is no need for ZWJ.) Note that in older versions of the digital text, this template was originally called "H" (=Hiriq for supplying the hiriq of the missing yod) instead of "מ:ירושלם", and this usage still appears in some of the chapters at Hebrew Wikisource.{{מ:ירושלם|משתנה1|משתנה2}}

This edition utilizes the special niqqud in Unicode for qamaz qatan, which distinguishes it from a normal qamaz. However, there are places where traditions differ as to whether a qamaz is "qatan" or not. In general, the Sephardic tradition has less of a tendency to voice qamaz qatan within certain grammatical forms, while theoretical grammar is more likely find qamaz qatan in those same forms. The latter has had a significant influence on the pronunciation of Modern Hebrew (including the reading of the Torah by speakers of Modern Hebrew). This template utilizes a minimum of two optional parameters, usually "ד=" (for the more theoretical grammar) and "ס=" (for the Sephardic tradition). In Wikisource the template is currently set to display "ד=" as the default, but that can easily be changed, and the future goal is to allow the user to set his own preference.{{מ:קמץ|ד=|ס=}}

General note: The masoretic tradition requires careful defined "song" styles for certain songs and lists that appear within the 21 "prose" books. In Wikisource, these texts are presented in two main parallel forms: (1) fully formatted songs created through the use of table code, (2) partially formatted songs created through the use of parashah spaces (see above). Neither of these two forms is accomplished through the use of additional templates beyond those already listed. However, at times the "song" format requires special alignment of the text immediately before and after the songs (and sometimes also for the first line of a song). This special requirement is tagged by the following two templates.

If a poetic verse naturally divides into an odd number of stiches (most commonly 3), then this template is inserted within the verse at an appropriate point, following a disjunctive cantillation, for an additional division that will allow the verse to be displayed in parallel columns (right facing left). This traditional form of display is facilitated by division of each verse into an even number of parts. The actual display of poetic verses at Wikisource does not employ this method, however, and so the template is currently set to show a simple space between words.
On rare occasion, when this template occurs immediately after {{מ:לגרמיה}}, it is set with an optional parameter like this: {{ר0|רווח=לא}}, indicating that no space is to be shown, because {{מ:לגרמיה}} already creates a space (two consecutive spaces between words would otherwise upset the aesthetics of the text).		{{ר0}}

In Job and Proverbs there are "titles" set in the center of a line. The phenomenon is not found in the rest of the masoretic bible, and it differs from the regular concept of parashah divisions. This template indicates that the "title" is meant to be centered, and its one parameter is the text of the "title" itself.
